# Quality Bar

Before finalizing, check:

- Did the answer treat the source as a proven viral reference rather than a flawed draft?
- If the source was an IG Reels / TikTok / Douyin link, was the actual video accessed or was an error reported immediately?
- Did it avoid hallucinating video content when transcript, visual frames, or accessible media were unavailable?
- Did it clearly separate direct evidence, secondary evidence, and inference when those confidence levels differ?
- If transcript text came from another report or partial extraction, did it label the confidence boundary instead of flattening it into fact?
- Did it clearly state which dependency stack was actually used in the run, such as `yt-dlp + ffmpeg + faster-whisper + OCR`?
- Did it analyze copy content, viral reason, and rhythm separately?
- Did it extract mechanism, not just meaning?
- When enough evidence existed, did it include a line-level or beat-level mapping of what the source lines are doing?
- When local video extraction was available, did it use a sufficiently dense visual pass instead of relying only on sparse screenshots?
- Did the report identify how long critical shots or subtitles stay visible, especially in the opening and CTA?
- Does the rewrite preserve the source video's hook mechanism and pacing curve?
- Does the rewrite adapt the mechanism without copying exact wording?
- Can a Taiwanese non-technical viewer understand and enjoy it?
- Can the production team shoot or edit from the output directly?
- If file tools are available, was a complete HTML report created without simplifying or deleting the analysis?
- Does the HTML report improve readability through layout, headings, tables, and execution checklists rather than reducing depth?
- Does the HTML follow the canonical visible section order rather than inventing a one-off structure?
- Did it avoid unsupported hard numbers, percentages, or performance claims that were not actually retrieved from evidence?
